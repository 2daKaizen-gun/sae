일본 자사개발 회사 취업용 포트폴리오. 다국어(日/英/韓).

- 협업 방식: `docs/collaboration.md` — 어떻게 함께 일하는가
- 작업 사이클: `docs/workflow.md` — PCTC(Plan→Code→Test→Commit)
- 인수인계: `docs/handoff.md` — 단계·세션 경계에서 맥락을 넘기는 템플릿 (최신 인수인계부터 이어서 시작)
- 데이터 모델: `docs/data-model.md` — SwiftData 스키마 ERD

## 헌법 요약

조항끼리 충돌하면 **번호가 낮은 조항이 우선**한다. (전문은 `docs/CONSTITUTION.md`)

**제품 원칙**
1. **측정 정확도는 타협 불가 (최상위)** — 타이밍 경로에 불확실한 지연 금지, 디스플레이·터치 지연 보정, 정확도는 테스트로 증명. 화려함이 정확도를 위협하면 정확도를 택한다.
2. **과학적 정직성** — 모든 지표는 출처를 댈 수 있어야. 冴え度 점수는 설명 가능해야. 과장·의학적 진단 흉내 금지.
3. **프라이버시는 기본값** — 건강 데이터 온디바이스 처리, 서버 미전송, 추적·광고 SDK 금지.
4. **차가운 데이터, 따뜻한 전달** — さえちゃん은 톤만 따뜻하게, 숫자는 왜곡 없이. 안내는 행동 가능하게.
5. **국제화 1급 시민** — 문자열 하드코딩 금지, 처음부터 String Catalog(`.xcstrings`), 3언어 동등.
6. **포트폴리오 품질** — 모든 커밋은 면접에서 설명 가능해야. 명확함 > 영리함. 핵심에는 테스트.
7. **네이티브 우선·단순함** — 애플 네이티브 스택, 외부 의존성 최소, MVP 스코프 사수, 미리 추상화 금지.

**협업 원칙 (내가 일하는 방식)**
8. **문서를 진실의 원천으로 (context rot 방지)** — 헷갈리면 기억이 아니라 문서·코드·git을 다시 본다(추측 금지). 확정된 사실 다시 캐묻지 않기. 중요한 결정은 그 자리에서 기록. 요약 이후에도 하던 일 이어가기. 과거 결정과 말없이 모순되지 않기.
9. **조급해하지 않기 (context anxiety 방지)** — 충분히 알면 행동. 질문은 답에 따라 행동이 바뀔 때만. "빨리 끝내려고" 범위 밖 작업 금지. 한 번에 하나씩 확인하고 넘어가기. 불확실하면 정직하게. 실패·미완은 그대로 보고.
